Sail #

Simple Api Interaction Language

Introduction

Sail (Simple Api Interaction Language) is a hypermedia language that aims to simplify the interaction with APIs. When simplifying the interaction, clients are less coupled and changes can be absorbed a bit easier.

Decoupling an API client from the data structure is highly difficult, if not impossible. Any attempt to generalize or standardize data structure could make things unnecessarily complex or lose alignment with the business domain. Instead, Sail focuses on simplifying the mechanism to request things to APIs either for getting information or triggering actions, in order to decupling the client as much as possible. For that matter, it uses primarily the hypermedia controls link and form.

It is important to clarify that Sail doesn't assume any architectural style, it could be used in RESTful API's as well as in any other approach where the format involved is JSON.

The format

Sail uses a JSON based format where anything that belongs to the language is marked with a # 

 { 
  "#links": []
 }
    
   {
    "#forms": []
   }
    
   {
    "#meta": []
   }

Links

Links in Sail have a very simple structure. At least an href and rel fields must be specified. All links must be in a #links section.

 "#links": [
       {
        "href":"http://api.com/products/1/reviews",
        "rel":"reviews"
       }          
   ]
     
   GET http://api.com/resources/1
