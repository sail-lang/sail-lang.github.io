Forms

Forms are the mechanism for decoupling actions like creation, modification or deletion. When using forms the client doesn't need to cope with URLs, HTTP verbs, headers and even in some cases with bodies. By default forms submmit a JSON body, but other body formats can be specified. All forms must be specified in a #form section. The fields id, action and method are required whereas the body is optional.

Immutable forms

The most useful forms are those that don't need to be filled. If the API can provide a form with all fields/inputs pre-set, the client only needs to submit the form without worrying about what is inside. When the API evolves to something that requires new or different fields that also can be pre-set, the client is not affected at all. This types of forms are called immutable and are specified with the field body. It is expected that the client doesn't change anything of what it is in the field body. 

"#forms": [
      {
        "id":"add",
        "action":"http://api.com/carts/23423",
        "method":"POST",
        "body": {
              "id":"324234"
            }
       }          
    ]
     
        POST http://api.com/carts/23423

	{
          "id":"324234"
        }

Regular forms

In several cases, forms can't be pre-set and some values must be provided by the client. Forms that have to be filled are called regular and the body is specified in the field inputs. Though this forms generates certain level of coupling, as the client needs to know what must be filled, it offers some advantages like decoupling part of the protocol specifics (URL and HTTP verb) and it advertises what the API is expecting as the body. Through the use of inputs and its variants (select, multiselect, etc) some things can be done in order to minimize the coupling as much as possible. 

 "#forms": [
     {
       "id":"add",
       "action":"http://api.com/products",
       "method":"POST",
       "inputs": [
             {
              "name":"name",
              "value":
             },
             {
              "name":"description",
              "value":
             }
          ]
      }          
   ]
     
        POST http://api.com/products

	{
         "name":"A product",
         "description":"A great product"
        }

There are tree types of inputs: simple, select and multiselect (but there is some work in progress to provide more alternatives). A simple input specifies a name and a value to be provided. It's the most basic type of input and the client must provide the value, although a default value can be specified. Select and multiselect offer the possibility to specify which values are allowed for that field. As their names suggest, a select input only allows one option to be selected whereas the multiselect allows multiple values to be selected at the same time.

"#forms": [
     {
       "id":"add",
       "action":"http://api.com/products",
       "method":"POST",
       "inputs": [
            {
              "name":"name",
              "value":
            },
            {
              "name":"color",
              "select":[
                 {
                     "title":"Red",
                     "value":"RED"
                 },
                 {
                     "title":"Green",
                     "value":"GREEN"
                 }
              ]
            },
            {
              "name":"categories",
              "multiselect":[
                 {
                     "title":"Audio",
                     "value":"AUDIO"
                 },
                 {
                     "title":"Technology",
                     "value":"TECH"
                 },
                 {
                     "title":"Books & Magazines",
                     "value":"BOOK"
                 }
              ]
             }
          ]
      }          
   ]
     
        POST http://api.com/products

	{
         "name":"Player",
         "color":"RED",
         "categories": ["AUDIO" ,"TECH"]
        }
